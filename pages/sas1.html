Dataset Description:¶

The file fred.sas7bat contains data on 143 bicycle rides taken in 2013 by Fred, an amateur athlete and cycling enthusiast. The dataset includes the following variables: date, the date on which the ride was take; time, the elapsed time (or duration) of the ride; distance, the length in miles of the ride; elgain, the total (not net) elevation gain during the ride; avgspd, Fred’s average speed (mi/hr) during the ride; maxspd, Fred’s maximum speed during the ride; avghr, Fred’s average heart rate during the ride; maxhr, Fred’s maximum heart rate during the ride; cals, the number of calories Fred burned during the ride; and strokes, the number of pedal strokes Fred made during the ride. These data were collected from Fred’s cycling computer. During the year Fred noticed some malfunctions in his cycling computer, so he knows that there are a few mistakes in the dataset as well as some missing values (but most of the missing values are for the variable strokes, which we are not going to use in this problem).
I coded a SAS program that sets the Fred dataset into a new dataset and create the following new variables: logdist, the natural logarithm of distance; elgrad, the elevation gradient equal to the ratio of elgain to distance (this variable measures how steep the bike route was); and I created an indicator called calgroup for whether the ride was after January 18, 2013. The reason for calgroup is that Fred’s cycling computer computes calories burned for each ride, but Fred had not entered his personal data (his weight, fitness level, etc.) into the computer until after the January 18 bike ride, so the calorie calculation was calibrated differently after that date as compared with the first few rides of 2013.

libname SASdata "/folders/myshortcuts/myFolders/Data Files";

************Prepare the new dataset*************;

proc format; *format calgroup;
  value cal_format 1='Before January 18' 2="After January 18";
run;


data sasdata.fred_new; 
  set sasdata.fred; *copy from the original dataset;
  
  if (date <= '18jan2013'd) 
      then calgroup=1; 
  else calgroup = 2;    *choose cal group based on date;
  
  format calgroup cal_format.;
  
  if calgroup=2 
     then calfixed=cals; 
  else calfixed=.;  
  
  if calgroup=1 
     then calnotfixed=cals; 
  else calnotfixed=.; 
  
  
  logdist=log(distance);
  
  elgrad=elgain/distance;
  
  format date mmddyy8. time time8.0;
  
  label avgspd="AverageSpeed" avghr="AverageHeartrate" cals="CaloriesBurned" time="Duration" distance="Distance" date="Date" elgrad="ElevationGradient" logdist="LogDistance" calfixed="CaloriesBurned"  calnotfixed="CaloriesBurned";
run;

(h) Does the fitted regression model suggest that Fred's average speed has gone up during the year? Does the regression model seem to fit these data well? Why or why not?
The fitted regression line is almost perfectly flat. We would have to look at the regression coefficients and their t-statistics to determine if the slope is significantly different from 0, but it appears not to be, suggesting that Fred’s pace has not increased during the year. However, a simple linear regression model doesn’t appear to fit these data well. The data seem to increase over time after mid-year, peaking in September or October. Then, in November and December Fred’s pace exhibits a steep decline. This is shown clearly by the loess line. Therefore, the regression line is somewhat misleading; it over-simplifies the pattern that the data actually exhibit.
